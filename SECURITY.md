# Security Policy

## Supported versions

This project moves quickly. Security fixes are prioritized for:

| Version | Supported |
| --- | --- |
| `main` | Yes |
| Latest public release line | Yes |
| Older releases | No |

## Reporting a vulnerability

Please **do not** open public GitHub issues for security vulnerabilities.

Use GitHub's private vulnerability reporting for this repository when it is available. If private reporting is not available in your environment, contact the repository maintainers through a private channel before disclosing details publicly.

When you report an issue, include as much of the following as you can:

- affected component or path
- affected version, release, or commit
- reproduction steps or proof of concept
- impact and attack scenario
- any mitigations or workarounds you found

## Disclosure

Please give maintainers a reasonable opportunity to validate and address the issue before public disclosure. Coordinated disclosure is preferred.
